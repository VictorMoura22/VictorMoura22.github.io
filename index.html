§ 2.6

Fundamentação Teórica

Teorema de Picard–Lindelöf
Existência, unicidade e convergência

Dado o PVI \(x' = F(t,x)\), \(x(t_0)=x_0\), com \(F\) contínua e Lipschitziana em \(x\), fixe \(\delta > 0\) tal que \(B_\delta(t_0)\times B_\delta(x_0)\subset U\). Define-se:

M(δ) = sup{ ‖F(t,x)‖ : |t−t₀| ≤ δ, ‖x−x₀‖ ≤ δ }
C(δ) = constante de Lipschitz de F em x sobre essa região
ε(δ) = min{ δ, δ/M(δ) } — raio garantido de solução

Para todo \(\varepsilon \leq \varepsilon(\delta)\) existe solução única \(x:(t_0-\varepsilon, t_0+\varepsilon)\to\mathbb{R}^d\), e os iterados de Picard convergem uniformemente (Teorema 2.12 de [1]).

Cota do erro — Corolário 2.14
Velocidade de convergência
‖γₙ − x‖∞ ≤ M(δ) · (C(δ)·ε)ⁿ / n!

O fatorial no denominador garante convergência para qualquer valor de \(C(\delta)\varepsilon\). Se \(C\varepsilon \leq 1\): a cota decresce desde o primeiro iterado. Se \(C\varepsilon > 1\): cresce até \(n \approx C\varepsilon\) e só então decai.

Para F(t,x) = x² — parâmetros ótimos
Maximização de ε(δ)
M(δ) = (1+δ)²
C(δ) = 2(1+δ)
ε(δ) = δ/(1+δ)²
δ* = 1 → ε* = 1/4, M* = C* = 4

Com \(C^*\varepsilon^* = 1\), a cota reduz-se a \(4/n!\), garantindo \(\|\gamma_8 - x\|_\infty \lesssim 10^{-4}\).

Sistema 2D — resultado central
C(δ) = 1 constante para x″ = −x + 2 sen t

Reescrevendo como sistema \((x_1,x_2)' = (x_2,\,-x_1+2\text{sen}\,t)\), a parte linear tem matriz \(A = \bigl(\begin{smallmatrix}0&1\\-1&0\end{smallmatrix}\bigr)\), que é uma rotação — isometria com \(\|A\|_2 = 1\). Logo \(C(\delta)=1\) para todo \(\delta\), tornando a cota \(M(\delta)\,\varepsilon^n/n!\) sem o fator \((C\varepsilon)^n\) que penalizava o caso escalar. A solução é inteira em \(\mathbb{R}\), sem nenhuma singularidade real.

Erro total — dois termos competidores
‖γₙʰ − x‖∞ ≤ M(δ)·(C(δ)·ε)ⁿ/n! + O(ε²·L⁻ᵖ) ↑ erro de convergência ↑ erro de quadratura p = 2 (trapézio) p = 1 (retângulo)

Aumentar \(N\) reduz o primeiro exponencialmente via o fatorial. Aumentar \(L\) reduz o segundo como \(L^{-p}\). O retângulo atinge um piso de erro irredutível \(\propto \varepsilon^2/L\) que não desaparece aumentando \(N\) — apenas aumentando \(L\).

EDO1.py · ε = 0,25 · L = 500 · N = 8
Operador de Picard — implementação inicial

Com \(C(\delta^*)\varepsilon^* = 1\), a cota \(\|\gamma_n-x\|_\infty\leq 4/n!\) decresce monotonamente desde o primeiro iterado. Para \(n=8\): \(4/8!\approx 9{,}9\times10^{-5}\).

EDO1.py 
import numpy as np

eps = 0.25;  L = 500;  N = 8;  x0 = 1.0
t = np.linspace(0, eps, L)

def F(t, x): return x**2

y = np.zeros((N+1, L));  y[0] = x0

for n in range(N):
    for k in range(L):
        intg = F(t[:k+1], y[n, :k+1])
        y[n+1, k] = x0 + np.trapezoid(intg, t[:k+1])
EDO2.py · ε = 0,5 · L = 2000 · N = 14
Convergência além de ε* — série geométrica

Para \(F=x^2\), os iterados coincidem com as somas parciais de \(x(t)=\sum_{k=0}^\infty t^k\), convergindo para todo \(|t|<1\) mesmo que o Teorema 2.12 só garanta \(\varepsilon\leq 0{,}25\). O trapézio atinge piso numérico \(\sim 10^{-7}\); o retângulo estabiliza em \(\sim 7\times10^{-4}\).

EDO3.py · ε = 0,85 · L = 10001 (ímpar) · N = 14
Integração bidirecional — piso irredutível do retângulo

\(L\) ímpar garante \(t_{i_0}=0\) exatamente (\(i_0=\lfloor L/2\rfloor\)). O erro global do retângulo em \([t,0]\) é \(\mathcal{O}(h)\cdot|t|=\mathcal{O}(\varepsilon^2/L)\), constante em \(N\). O trapézio converge como \(\mathcal{O}(\varepsilon^3/L^2)\) — duas ordens a mais em \(L\).

EDO4.py · ε = 0,99 · L = 20001 · N = 30
Acumulação incremental — O(NL²) → O(NL)

A recorrência \(I_k^+=I_{k-1}^++(f_{k-1}+f_k)h/2\) reduz o custo de \(\approx 3\times10^9\) para \(\approx 1{,}2\times10^6\) operações — fator \(2500\times\). O parâmetro ótimo \(\delta^*=1\) é o único máximo de \(\varepsilon(\delta)=\delta/(1+\delta)^2\).

EDO5.py · ε = 0,947 · L = 20001 · N = 14 · d = 2
Tensor (N+1)×L×2 — C(δ) = 1 exato

A matriz \(A=\bigl(\begin{smallmatrix}0&1\\-1&0\end{smallmatrix}\bigr)\) é isometria (\(\|A\|_2=1\)), logo \(C(\delta)=1\) para todo \(\delta\). Com \(C\varepsilon=1\), a cota decai como \(M(\delta^*)/n!\) desde o primeiro iterado. Solução inteira: sem limitação de domínio análoga ao teto \(|t|<1\).

Otimização — fator 2500×
O(NL²) → O(NL)

A versão direta recalcula \(\int_{t_{i_0}}^{t_k}\) integralmente a cada ponto: \(\sum_{k=i_0}^{L-1}(k-i_0)=\mathcal{O}(L^2)\) por iterado.

A acumulação incremental usa:

I⁺ₖ = I⁺ₖ₋₁ + (fₖ₋₁ + fₖ)/2 · h [trapézio]
I⁻ₖ = I⁻ₖ₊₁ + (fₖ + fₖ₊₁)/2 · h [negativo]

Cada passo custa Θ(1) → custo total O(NL). Para N=30, L=20001: ≈3×10⁹ → ≈1,2×10⁶ operações.
