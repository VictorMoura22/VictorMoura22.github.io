§ 2.6

Fundamentação Teórica

Teorema de Picard–Lindelöf
Existência, unicidade e convergência

Dado o PVI \(x' = F(t,x)\), \(x(t_0)=x_0\), com \(F\) contínua e Lipschitziana em \(x\), fixe \(\delta > 0\) tal que \(\bar{B}_\delta(t_0)\times \bar{B}_\delta(x_0)\subset \mathcal{U}\). Define-se:

M(δ) = sup{ ‖F(t,x)‖ : |t−t₀| ≤ δ, ‖x−x₀‖ ≤ δ }
C(δ) = constante de Lipschitz de F em x sobre essa região
ε(δ) = min{ δ, δ/M(δ) } — raio garantido de solução

Para todo \(\varepsilon \leq \varepsilon(\delta)\) existe solução única \(x:(t_0-\varepsilon, t_0+\varepsilon)\to\mathbb{R}^d\), e os iterados de Picard \(\gamma_0 \equiv x_0\), \(\gamma_{n+1}(t) = x_0 + \int_{t_0}^t F(s,\gamma_n(s))\,ds\) convergem uniformemente (Teorema 2.12 de Viana & Espinar).

Cota do erro — Corolário 2.14
Velocidade de convergência
‖γₙ − x‖∞ ≤ M(δ) · (C(δ)·ε)ⁿ / n!

O fatorial no denominador garante convergência para qualquer \(C\varepsilon\). Se \(C\varepsilon \leq 1\): a cota decresce desde o primeiro iterado. Se \(C\varepsilon > 1\): cresce até \(n \approx C\varepsilon\) e só então decai — o que ocorre no Objetivo 4 com \(\varepsilon_{\mathrm{num}}=0{,}99\) e \(C\varepsilon \approx 3{,}96\).

Para F(t,x) = x² — parâmetros ótimos
Maximização de ε(δ)
M(δ) = (1+δ)²
C(δ) = 2(1+δ)
ε(δ) = δ/(1+δ)²
δ* = 1 → ε* = 1/4, M* = C* = 4

Com \(C^*\varepsilon^* = 1\), a cota reduz-se a \(4/n!\), garantindo \(\|\gamma_8 - x\|_\infty \lesssim 9{,}9\times10^{-5}\). O valor \(\delta^*=1\) é o único máximo de \(\varepsilon(\delta)=\delta/(1+\delta)^2\), obtido anulando a derivada: \((1-\delta)/(1+\delta)^3 = 0\).

Sistema 2D — Objetivo 5
C(δ) = 1 constante — isometria

Para \(x'' = -x + 2\sin t\), o sistema equivalente é \((x_1,x_2)' = (x_2,\,-x_1+2\sin t)\) com parte linear \(A = \bigl(\begin{smallmatrix}0&1\\-1&0\end{smallmatrix}\bigr)\). Como \(A\) é uma rotação (isometria com \(\|A\|_2 = 1\)), a constante de Lipschitz é \(C(\delta) = 1\) para todo \(\delta\). Consequentemente, com \(C\varepsilon = 1\) a cota \(M(\delta)\,\varepsilon^n/n!\) decresce monotonamente desde o primeiro iterado, sem o período de crescimento inicial que afeta o Objetivo 4. Além disso, a solução \(x_1(t) = (1-t)\cos t + \sin t\) é inteira em \(\mathbb{R}\), sem singularidades reais.

Erro total — dois termos competidores
‖γₙʰ − x‖∞ ≤ M(δ)·(C(δ)·ε)ⁿ/n! + O(ε²·L⁻ᵖ) ↑ erro de convergência ↑ erro de quadratura p = 2 (trapézio, O(h²) por passo) p = 1 (retângulo, O(h) por passo)

Aumentar \(N\) reduz o primeiro exponencialmente via o fatorial; aumentar \(L\) reduz o segundo como \(L^{-p}\). O retângulo atinge um piso de erro irredutível \(\propto \varepsilon^2/L\) que não desaparece aumentando \(N\) — apenas aumentando \(L\) ou diminuindo \(\varepsilon\). O trapézio converge duas ordens a mais em \(L\), tornando-o sempre preferível.

Viana & Espinar § 2.6 — cinco objetivos progressivos
1

Implementação básica concluído

Operador de Picard como recorrência matricial em \([0,\varepsilon^*]\) com \(\varepsilon^*=0{,}25\). Primeira validação contra a solução exata \(x(t)=1/(1-t)\). Com \(C\varepsilon^*=1\), a cota \(4/n!\) decresce monotonamente desde o 1.º iterado.

2

Trapézio vs. Retângulo concluído

Comparação dos dois integradores em \([0,\,0{,}5]\). Os iterados coincidem com as somas parciais de \(\sum t^k\), explicando a convergência além de \(\varepsilon^*\) para todo \(|t|<1\). O trapézio atinge piso numérico \(\sim 10^{-7}\); o retângulo estabiliza em \(\sim 7\times10^{-4}\).

3

Domínio simétrico \((-\varepsilon,\varepsilon)\) concluído

Integração bidirecional a partir de \(t_0=0\). \(L\) ímpar garante \(t_{i_0}=0\) exatamente (\(i_0=\lfloor L/2\rfloor\)). Piso irredutível do retângulo no extremo esquerdo \(\propto\varepsilon^2/L\) identificado e explicado: o erro acumulado integrando com pontos à esquerda não se cancela aumentando \(N\).

4

Domínios maiores — análise teórica + otimização concluído

Cálculo de \(M(\delta)\), \(C(\delta)\), \(\delta^*=1\), \(N_{\max}(\tau)\). Redução de complexidade \(\mathcal{O}(NL^2)\to\mathcal{O}(NL)\) via acumulação incremental — fator \(\approx 2500\times\) para \(N=30\), \(L=20001\) (de \(\approx 3\times10^9\) para \(\approx 1{,}2\times10^6\) operações).

5

Sistema 2D — \(d=2\) concluído

Dimensões \((N+1)\times L\times 2\). \(C(\delta)=1\) exato via norma espectral de \(A=\bigl(\begin{smallmatrix}0&1\\-1&0\end{smallmatrix}\bigr)\). Solução inteira \(x_1(t)=(1-t)\cos t+\sin t\).

EDO1.py · ε = 0,25 · L = 500 · N = 8
Operador de Picard — implementação inicial

O domínio \([0, \varepsilon^*=0{,}25]\) maximiza \(\varepsilon(\delta)=\delta/(1+\delta)^2\) em \(\delta^*=1\). Com \(C(\delta^*)\varepsilon^* = 1\), a cota \(\|\gamma_n-x\|_\infty\leq 4/n!\) decresce monotonamente desde o primeiro iterado. Para \(n=8\): \(4/8!\approx 9{,}9\times10^{-5}\). A convergência observada é tipicamente melhor, pois a cota usa o supremo sobre toda a bola \(B_{\delta^*}(x_0)\) e não apenas sobre a trajetória da solução.

EDO2.py · ε = 0,5 · L = 2000 · N = 14
Convergência além de ε* — série geométrica

Para \(F=x^2\), os iterados de Picard coincidem exatamente com as somas parciais de \(x(t)=\sum_{k=0}^\infty t^k = 1/(1-t)\), cujo raio de convergência é \(|t|<1\). Portanto, para qualquer \(\varepsilon<1\), os iterados convergem para a solução exata, embora o Teorema 2.12 só garanta isso para \(\varepsilon\leq 0{,}25\). Este é um dos pontos matematicamente mais elegantes: o método de Picard constrói, iterado a iterado, os coeficientes da expansão em série de Taylor da solução.

O trapézio atinge piso numérico \(\sim 10^{-7}\); o retângulo estabiliza em \(\sim 6{,}93\times10^{-4}\) — piso irredutível que não diminui aumentando \(N\).

EDO3.py · ε = 0,85 · L = 10001 (ímpar) · N = 14
Integração bidirecional — piso irredutível do retângulo

Para \(t<0\), a equação integral exige \(\gamma_{n+1}(t) = x_0 - \int_t^0 F(s,\gamma_n(s))\,ds\). O algoritmo divide-se em dois ramos a partir de \(i_0=\lfloor L/2\rfloor=5000\).

O erro global do retângulo ao integrar sobre \([t,\,0]\) é \(\mathcal{O}(h)\cdot|t| = \mathcal{O}(\varepsilon^2/L)\), constante em \(N\). O trapézio converge como \(\mathcal{O}(\varepsilon^3/L^2)\) — duas ordens a mais em \(L\). Os decaimentos observados no gráfico de erros são pontos onde a curva iterada toca a solução exata antes de cruzá-la no sentido da convergência.

EDO4.py · ε = 0,99 · L = 20001 · N = 30
Acumulação incremental — O(NL²) → O(NL)

Na implementação direta, o custo de calcular \(\gamma_{n+1}(t_k)\) é \(\Theta(k-i_0)\) operações, totalizando \(\mathcal{O}(L^2)\) por iterado. A recorrência incremental \(I_k^+ = I_{k-1}^+ + (f_{k-1}+f_k)\,h/2\) custa \(\Theta(1)\) por passo, reduzindo para \(\mathcal{O}(L)\).

Para \(N=30\), \(L=20001\): \(\approx3\times10^9\to\approx1{,}2\times10^6\) operações — fator \(2500\times\).

Com \(\varepsilon_{\mathrm{num}}=0{,}99\) e \(C\varepsilon\approx3{,}96>1\), os iterados pioram antes de convergir (os primeiros \(\approx 4\) iterados crescem), consistentemente com a Observação 2.14. A convergência ocorre via a série geométrica, que converge para todo \(|t|<1\).

EDO5.py · ε = 0,947 · L = 20001 · N = 14 · d = 2
Dimensões (N+1)×L×2 — C(δ) = 1 exato

A adaptação para \(d=2\) é mínima estruturalmente: os arrays ganham uma dimensão extra e o NumPy aplica a recorrência elemento a elemento sem alteração de código. \(F(t,\mathbf{x}) - F(t,\mathbf{y}) = (x_2-y_2,\,-(x_1-y_1))^T\), portanto \(\|F(t,\mathbf{x})-F(t,\mathbf{y})\| = \|\mathbf{x}-\mathbf{y}\|\) e \(C(\delta)=1\) exato para todo \(\delta\).

Com \(C\varepsilon=1\), a cota \(M(\delta^*)/n!\) decresce desde o 1.º iterado — sem o período inicial de piora do Objetivo 4. A solução \(x_1(t)=(1-t)\cos t+\sin t\) é inteira em \(\mathbb{R}\): sem limitação de domínio análoga ao teto \(|t|<1\).

EDO1.py · ε=0,25 · L=500 · N=8 ⬇ Download 
import numpy as np

eps = 0.25;  L = 500;  N = 8;  x0 = 1.0
t = np.linspace(0, eps, L)

def F(t, x): return x**2

y = np.zeros((N+1, L));  y[0] = x0

for n in range(N):
    for k in range(L):
        intg = F(t[:k+1], y[n, :k+1])
        y[n+1, k] = x0 + np.trapezoid(intg, t[:k+1])

A matriz y[n,k] armazena \(\gamma_n(t_k)\). Para cada \(t_k\), integra \(F(s,\gamma_n(s))\) de \(0\) a \(t_k\) pela regra do trapézio sobre \(k+1\) pontos. Complexidade \(\mathcal{O}(NL^2)\) pois cada ponto recalcula toda a integral acumulada.

EDO2.py · trecho das duas regras ⬇ Download 
h = t[1] - t[0]

# Trapézio
for n in range(N):
    for k in range(L):
        intg = F(t[:k+1], y_tp[n, :k+1])
        y_tp[n+1, k] = x0 + np.trapezoid(intg, t[:k+1])

# Retângulo — pontos à esquerda de cada subintervalo
for n in range(N):
    for k in range(L):
        intg = F(t[:k+1], y_rt[n, :k+1])
        y_rt[n+1, k] = x0 + np.sum(intg[:-1]) * h

intg[:-1] seleciona os \(k\) pontos à esquerda de cada subintervalo, implementando a regra do retângulo \(\sum f_{j-1}\,h\).

EDO3.py · integração bidirecional ⬇ Download 
i0 = L // 2   # índice de t = 0

for n in range(N):
    for k in range(L):
        if k > i0:      # t > 0: integral positiva
            intg         = F(t[i0:k+1], y_tp[n, i0:k+1])
            y_tp[n+1, k] = x0 + np.trapezoid(intg, t[i0:k+1])
        elif k < i0:   # t < 0: sinal negativo
            intg         = F(t[k:i0+1], y_tp[n, k:i0+1])
            y_tp[n+1, k] = x0 - np.trapezoid(intg, t[k:i0+1])
        else:
            y_tp[n+1, k] = x0

O sinal negativo em x0 - trapezoid(...) implementa \(\gamma_{n+1}(t) = x_0 - \int_t^0 F(\cdot)\,ds\) para \(t<0\). L ímpar garante \(t_{i_0}=0\) exato.

EDO4.py · acumulação incremental O(NL) ⬇ Download 
for n in range(N):
    f_tp = F(t, y_tp[n])     # avalia F uma vez em todos os pontos

    I_tp_pos = np.zeros(L)
    for k in range(i0 + 1, L):           # direção positiva
        I_tp_pos[k] = I_tp_pos[k-1] + (f_tp[k-1]+f_tp[k])/2.0 * h

    I_tp_neg = np.zeros(L)
    for k in range(i0 - 1, -1, -1):      # direção negativa
        I_tp_neg[k] = I_tp_neg[k+1] + (f_tp[k]+f_tp[k+1])/2.0 * h

    y_tp[n+1, i0+1:] = x0 + I_tp_pos[i0+1:]
    y_tp[n+1, :i0]   = x0 - I_tp_neg[:i0]
    y_tp[n+1, i0]    = x0

A recorrência \(I_k^+ = I_{k-1}^+ + (f_{k-1}+f_k)h/2\) custa \(\Theta(1)\) por passo, reduzindo o custo total de \(\approx3\times10^9\) para \(\approx1{,}2\times10^6\) operações (fator 2500×) para \(N=30\), \(L=20001\).

EDO5.py · sistema 2D - Dimensões (N+1)×L×2 ⬇ Download 
def F(t, x):            # x tem forma (L, 2)
    dx1 = x[:, 1]
    dx2 = -x[:, 0] + 2.0 * np.sin(t)
    return np.column_stack([dx1, dx2])

y_tp = np.zeros((N+1, L, 2))
y_tp[0, :, :] = x0      # x0 = array([1., 0.])

I_tp_pos = np.zeros((L, 2))
for k in range(i0 + 1, L):
    I_tp_pos[k] = I_tp_pos[k-1] + (f_tp[k-1]+f_tp[k]) / 2.0 * h

A única modificação estrutural em relação ao EDO4: arrays ganham dimensão extra. O NumPy aplica a recorrência elemento a elemento para ambas as componentes sem alteração de código. A condição inicial passa de escalar para vetor.

Síntese

Conclusões

Resultado central
O papel de C(δ) na estruturação do método

O desempenho do método de Picard depende diretamente da estrutura algébrica do campo \(F\). A constante de Lipschitz \(C(\delta)\) determina tanto o tamanho do domínio acessível quanto a velocidade de convergência, via a cota \(M(\delta)(C(\delta)\varepsilon)^n/n!\). Quanto mais próximo de linear for \(F\) em \(x\), mais favorável é o método.

1

Trapézio sempre preferível ao retângulo

O trapézio tem ordem \(\mathcal{O}(h^2)\) frente ao \(\mathcal{O}(h)\) do retângulo, resultando num piso de erro irredutível mais baixo que não pode ser eliminado aumentando \(N\). No Objetivo 2, o piso do retângulo estabiliza em \(\approx 6{,}93\times10^{-4}\) enquanto o trapézio atinge \(\sim 10^{-7}\) — quatro ordens de diferença para os mesmos parâmetros.

2

Convergência além do raio teórico para F = x²

O Teorema 2.12 garante convergência apenas para \(\varepsilon\leq\varepsilon^*=0{,}25\), mas os iterados convergem numericamente em todo \(|t|<1\). Isso ocorre porque os iterados coincidem com as somas parciais da série geométrica \(\sum t^k\), cujo raio de convergência é \(|t|<1\). Para \(|t|>1\) a divergência é verificável executando o Código 4 com \(\varepsilon>1\).

3

Otimização computacional — fator 2500×

A substituição da integração direta pela acumulação incremental \(I_k^+ = I_{k-1}^+ + (f_{k-1}+f_k)h/2\) reduz a complexidade de \(\mathcal{O}(NL^2)\) para \(\mathcal{O}(NL)\), tornando viável trabalhar com \(L=20001\) pontos. Para \(N=30\): de \(\approx3\times10^9\) para \(\approx1{,}2\times10^6\) operações.

4

Sistema 2D intrinsecamente mais favorável

Com \(C(\delta)=1\) constante (parte linear é isometria), o produto \(C\varepsilon=1\) garante que a cota decai monotonamente desde o primeiro iterado, sem o período de piora inicial do Objetivo 4 onde \(C\varepsilon\approx3{,}96\). Além disso, a solução é inteira em \(\mathbb{R}\): sem limitação de domínio análoga ao teto \(|t|<1\).

5

Equilíbrio entre N e L

Os dois termos do erro total têm dependências opostas: aumentar \(N\) reduz o erro de convergência exponencialmente via o fatorial; aumentar \(L\) reduz o erro de quadratura como \(L^{-p}\). O valor \(L\) mínimo efetivo para equilíbrio satisfaz \(\varepsilon^2 L^{-p} \sim M(\delta^*)(C\varepsilon^*)^N/N!\). Na prática, para \(N=14\) e trapézio, \(L\sim2000\) já equilibra os dois termos.
